## 🌟 Como Funciona a Trilha de Estudos
Esta trilha está organizada em **fases**, e cada uma aborda um conjunto de conceitos, ferramentas e práticas essenciais para a Engenharia de Dados e seu desenvolvimento aqui na Cognitivo.

### 🛠️ Estrutura das Fases:

- 📚 **Materiais teóricos**: artigos, vídeos e documentações para leitura e estudo.
- 🛠️ **Atividades práticas**: projetos com níveis de dificuldade graduais crescentes para consolidar o aprendizado.
- 💡 **Dicas e recursos adicionais**: para aprofundar o conhecimento.

A sequência das fases é pensada para ser progressiva:

- 🧩 Primeiro, você aprenderá os fundamentos e conceitos básicos.
- 🚧 Depois, aplicará esses conceitos em projetos mais complexos e desafiadores.

**Importante:**

- 🔍 Você é encorajado(a) a explorar e pesquisar além dos materiais fornecidos.
- 🌱 Este é um ambiente seguro para aprender, errar e crescer, então aproveite ao máximo cada desafio.

## 🏗️ Como realizar os projetos práticos

### **Amazon Web Services (AWS) e ambiente de DEV da Cognitivo:**

1. 💸 Tudo que é criado na Cloud gera custos em dólar (embora alguns recursos possuam free-tiers). Não crie nada em sua conta pessoal da AWS.
2. 🌐 Nós da Cognitivo contamos com uma conta especial de desenvolvimento. Para obter acesso, comunique-se no canal oficial da Engenharia, e o time de infraestrutura providenciará os acessos para você.

### **Seu repositório individual no GitHub para a trilha:**

Após criar sua conta no GitHub (associada à Cognitivo), ajudaremos você a configurar um repositório específico para armazenar seus projetos da trilha.

#### 📁 **Estrutura básica:**

1. Clone o repositório para sua máquina local.
2. Crie uma nova branch chamada **`develop`** para realizar o trabalho. Tudo que fizer em código deve ser nessa branch.

#### **Como Estruturar os Projetos no Repositório:**

- Para cada projeto prático, crie uma pasta com o nome do projeto.
  - 🗂️ Exemplo: Se você está na **Fase 1 - Projeto 1**, crie a pasta `Projeto 1` dentro do repositório. Adicione todos os códigos, arquivos e documentações nessa pasta.

### **Abertura de Pull Requests:**

1. 📝 Após finalizar o projeto, abra uma Pull Request detalhada da branch `develop` para a branch `main`.
2. Inclua **prints de telas**, **logs de execução** e informações relevantes para facilitar a revisão do código.
3. ✔️ A revisão determinará se o projeto atendeu às expectativas.

### **Como saber se devo progredir nos módulos?**

1. 🚶‍♂️ Finalize o estudo teórico do próximo módulo enquanto o desafio anterior está em revisão.
2. ⏳ Não se apresse! O objetivo é construir uma forte base de fundamentos.
